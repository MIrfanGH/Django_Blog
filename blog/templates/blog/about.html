About This Project

A production Mini-SaaS platform built from scratch. The interesting parts aren't the features; they're the failure modes I had to think through: cache stampedes, duplicate webhook delivery, worker crashes mid-task, stale summaries after content changes. Each one has a deliberate fix built in.

The full stack runs on AWS with a custom VPC, Auto Scaling Group, and ALB — stateless by design, scalable by default.

Key Features

  • Full user authentication with email-based password reset
  • Role-based access control (Reader / Author / Admin) via custom Django mixins
  • Blog CRUD with pagination and select_related query optimization
  • Redis caching with signal-driven invalidation on every post change
  • Async Celery pipelines with exponential backoff and distributed lock lifecycle management
  • Stripe donations with HMAC-verified webhooks and two-layer idempotency
  • AI summarization — async, cache-aside with DB fallback, stampede-protected
  • Profile image processing via Celery: S3 download → resize → re-upload, non-blocking
  • Stateless design enabling true horizontal scaling behind the ALB
  • AWS: EC2, RDS (private subnet), S3, VPC, Auto Scaling Group, ALB, Cloudflare Full Strict SSL

Technology Stack

  • Django 5.2 (Class-Based Views, Signals, ORM, Authentication)
  • PostgreSQL (AWS RDS, private subnet) · SQLite (development)
  • Redis — Django cache (DB0) + Celery broker (DB1) + distributed locking
  • Celery 5.5 — async task queue with exponential backoff and Beat scheduler
  • Stripe API — Checkout Sessions, HMAC-verified webhooks, idempotent event handling
  • LLM APIs — AI-powered post summarization with cache-aside retrieval
  • AWS EC2, S3, RDS, VPC, ASG, ALB
  • Nginx (reverse proxy, Unix socket) · Gunicorn (WSGI server) · systemd
  • Cloudflare Full (Strict) SSL — HTTPS enforced end to end
  • Pillow — async profile image resizing via Celery
